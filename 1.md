
🛠️ 技术选型与工具链
建议使用 Python 3.10+。理由：Python 在处理复杂逻辑、机器学习集成和快速 UI 开发上效率极高，且有强大的第三方库支持可视化，非常适合 2 人团队在 2 个月内完成高质量大作业。
 * GUI 框架：PyQt6 或 PySide6（支持现代、丝滑的界面设计）。
 * 计算与统计：NumPy（处理字节流效率高）、Pandas（统计压缩率数据）。
 * 机器学习（选做）：Scikit-learn（实现轻量级算法选择器）。
 * 绘图/可视化：Matplotlib 或 PyQtGraph（实时折线图）；Graphviz（可视化 Huffman 树）。
🚀 0-1 起点：构建核心认知
在开始写代码前，你们需要完成以下三步，这是项目的底座：
 * 字节流思维：网页资源（HTML, JPG, JS）在底层都是二进制数据。你们需要掌握如何用 Python 的 rb (read binary) 模式读取文件，并将比特流（bitstream）打包成字节。
 * Huffman 进阶：掌握如何将 Huffman 树序列化存入压缩包头，否则解压端无法还原。
 * 字典压缩初探 (LZ77)：Huffman 对字符频率敏感，但网页代码有大量的重复模式（如 <div>, class=, function）。LZ77 算法通过“滑动窗口”寻找重复子串，它是 Gzip 的核心。
✨ 亮点功能：如何脱颖而出
要让项目在答辩时惊艳，可以从以下三个维度进行“模块化”拔高：
1. 智能算法调度器 (Smart Dispatcher) — AI 加持
想法确认：用机器学习是好主意，但不要用深层神经网络，建议用决策树 (Decision Tree)。
 * 实现细节：
   * 特征提取：分析文件的熵值（Entropy）、重复率、文件后缀、文件大小。
   * 逻辑：模型根据特征自动决定：文本类使用 LZ77 + Huffman；图片类使用 离散余弦变换 (DCT) 或 颜色量化；已压缩过的文件（如 JPG）则直接跳过或采用无损存储模式。
   * 意义：体现了 F4 自适应压缩优化的要求。
2. 差分/增量压缩 (Incremental Compression)
 * 实现细节：模拟网页更新场景。如果用户只改了 HTML 里的一个标题，系统不重新压缩全量文件，而是计算新旧文件的 Diff，只压缩并传输差异部分。这在实时渲染和网络传输优化中非常有竞争力。
3. 深度算法可视化 (Educational Visualization)
这是加分大项，满足 F7 可视化分析的要求。
 * Huffman 动态构建：点击“压缩”后，界面实时演示从森林合并成树的动画。
 * LZ77 滑动窗口演示：用两个不同颜色的方框在文本流上移动，展示“匹配缓冲区”寻找重复串的过程。
📦 模块化设计与算法细节
M1: 资源适配与预处理模块 (Adaptor)
 * 功能：识别文件类型。
 * 数据结构：HashMap (字典)，存储后缀名与压缩策略的映射关系。
M2: 混合压缩引擎 (Hybrid Engine)
 * 核心算法：
   * LZ77：负责“去冗余”。使用滑动窗口 (Sliding Window) 数据结构 。
   * Huffman：负责“变长编码”。使用优先级队列 (Priority Queue) 构建最优二叉树。
 * 公式应用：计算压缩率 Compression\ Ratio = \frac{Original\ Size - Compressed\ Size}{Original\ Size} \times 100\%。
M3: 传输模拟与实时渲染 (Network Sim)
 * 功能：模拟 4G/5G/WiFi 环境。
 * 算法细节：
   * 引入 Time 延迟模拟器。
   * 计算公式：Transmission\ Time = \frac{Compressed\ Size}{Bandwidth} + \text{Latency}。
   * 对比压缩前后在不同网络下的页面加载时长。
